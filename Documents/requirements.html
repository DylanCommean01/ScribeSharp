ScribeSharp Requirements

Team members

Rashaad Washington and Dylan Commean

Purpose

To provide clarity and integration of note taking within a classroom setting.

Requirements

  1. Name: select_user_profile
    Type: functional
    Summary: Menu option that allows users to select whether they are logged in. If they would like an account, allow them to register.

  2. Name: register_user
    Type: functional
    Summary: User is prompted to register a profile to the website. Adds that user to the database.

  3. Name: login_user
    Type: functional
    Summary: Users are prompted to login using their username and password. It checks the database for the valid information and tells the user to login again if the login information is not found or incorrect.

  4. Name: select_subject
    Type: functional
    Summary: Allows the user to select which subject widget in a list of Math or Computer Science they would like to implement in their notes.

  5. Name: select_application
    Type: functional
    Summary: Allows the user to select which application widget to use within certain subjects.

  6. Name: logout_button
    Type: functional
    Summary: Permits the user the ability to log their account out. Will force program to return to original state.

  7. Name: previous_state(back)
    Type: functional
    Summary: Allows the user to go back to a previous state. If at the homescreen, the button disappears.

  8. Name: user_notebook
    Type: functional
    Summary: Displays the user's past collection of notes.

  9. Name: user_save_notes
    Type: functional
    Summary: Users can save their notes to the database.

  10. Name: join_user_to_class
    Type: functional
    Summary: Users can join a note session hosted by a teacher.

  11. Name: direct_message
    Type: functional
    Summary: Users will be able to chat one-to-one with other users in the same class.

  12. Name: share_notes
    Type: functional
    Summary: Users can share and display their notes for other users in the same class.

  13. Name: free_note
    Type: functional
    Summary: All users have access to this. Allows the user to write notes. Functionality for the subject_widget is unavailable.

  14. Name: host_class
    Type: functional
    Summary: Teachers/Educators can host a session of notes for students to join. The user can import their own notes with extensions: .ppt, .doc, .pdf, and more.

  15. Name: internet_connection
    Type: constraint
    Summary: Users should have a connection to the internet.

  16. Name: reliability
    Type: constraint
    Summary: Functionality should be online at all times and rate of failure should be minimal.

  17. Name: platform_os
    Type: constraint
    Summary: The target platform operating system is Windows 10.

  18. Name: platform_resolution
    Type: constraint
    Summary: Must work on 4K monitors.

  19. Name: login_info_secure
    Type: risk
    Summary: User information shall be stored and read in a secure fashion.
